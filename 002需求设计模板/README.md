# 需求设计方法论与模板库

本目录包含完整的需求设计方法论、提示词模板和最佳实践指南。

## 📁 文件结构

```
需求设计模板/
├── README.md                          # 本文件
├── 01-方法论.md                       # 需求设计核心原则和方法论
├── 02-提示词模板集.md                 # 6个核心提示词模板
├── 03-检查清单.md                     # 需求文档质量检查清单
├── 04-常见陷阱.md                     # 常见错误和避免方法
├── 05-快速上手指南.md                 # 2小时完成需求文档的步骤
├── 06-示例对比.md                     # 好坏需求描述对比
└── templates/                         # 可直接使用的模板文件
    ├── 项目启动模板.md
    ├── 功能需求模板.md
    ├── 技术架构模板.md
    ├── 阶段规划模板.md
    ├── 风险评估模板.md
    └── 成本预估模板.md
```

## 🚀 快速开始

### 新项目需求设计（推荐流程）

1. **阅读方法论**（10分钟）
   - 打开 `01-方法论.md` 了解核心原则

2. **使用项目启动模板**（5分钟）
   - 复制 `templates/项目启动模板.md`
   - 填入项目基本信息
   - 使用AI生成初稿

3. **细化功能需求**（每个功能10分钟）
   - 使用 `templates/功能需求模板.md`
   - 为每个核心功能编写详细说明

4. **设计技术架构**（30分钟）
   - 使用 `templates/技术架构模板.md`
   - 完成技术选型和数据模型设计

5. **制定实施计划**（20分钟）
   - 使用 `templates/阶段规划模板.md`
   - 划分4个阶段的详细计划

6. **评估风险和成本**（30分钟）
   - 使用 `templates/风险评估模板.md`
   - 使用 `templates/成本预估模板.md`

7. **质量检查**（10分钟）
   - 使用 `03-检查清单.md` 自查

**总耗时：约2小时完成高质量需求文档**

## 📚 文档说明

### 01-方法论.md
包含需求设计的4大核心原则：
- 用户驱动原则
- 分层设计原则（7个层面）
- 阶段化实施原则
- 可验证性原则

### 02-提示词模板集.md
包含6个核心提示词模板：
1. 项目启动提示词
2. 功能需求细化提示词
3. 技术架构设计提示词
4. 阶段规划提示词
5. 风险评估提示词
6. 成本预估提示词

### 03-检查清单.md
包含4类检查项：
- 完整性检查（8项）
- 可行性检查（5项）
- 可验证性检查（5项）
- 优先级检查（4项）

### 04-常见陷阱.md
列举5个常见错误及避免方法：
- 需求过于抽象
- 技术选型缺乏依据
- 阶段划分不合理
- 忽略非功能需求
- 成本预估过于乐观

### 05-快速上手指南.md
7步完成需求文档的详细步骤和时间分配

### 06-示例对比.md
通过实际案例对比好坏需求描述的差异

## 🎯 适用场景

### ✅ 适合使用本模板的项目
- 基于mksaas-template的SaaS项目
- 需要快速MVP验证的创业项目
- 2-4周开发周期的中小型项目
- 需要分阶段迭代的产品

### ⚠️ 不太适合的场景
- 超大型企业级系统（需要更详细的文档）
- 纯研究型项目（不需要商业化考虑）
- 已有成熟需求流程的团队（可能有冲突）

## 💡 使用建议

### 对于个人开发者
- 重点使用"项目启动模板"和"快速上手指南"
- 可以简化风险评估和成本预估部分
- 专注于MVP阶段的快速验证

### 对于小团队（2-5人）
- 完整使用所有模板
- 团队一起评审需求文档
- 每周回顾实施进度和风险

### 对于AI辅助开发
- 直接复制提示词模板给AI
- 让AI生成初稿后人工审核
- 使用检查清单确保质量
