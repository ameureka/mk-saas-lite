# 项目启动提示词模板

> 复制下面的内容，填入你的项目信息，发送给AI生成需求文档初稿

---

我要设计一个[项目类型]的需求文档，请帮我按照以下结构完成：

## 背景信息
- **项目名称**：[填写项目名称，例如：AI智能PPT模板推荐平台]
- **核心问题**：[要解决的痛点，例如：用户需要花费大量时间浏览和筛选PPT模板，传统关键词搜索无法理解用户真实需求]
- **目标用户**：[用户画像，例如：职场人士（工作汇报）、教师（教学课件）、学生（学术答辩）]
- **竞品参考**：[类似产品，例如：第一PPT（1ppt.com）、包图网（ibaotu.com）]
- **技术基础**：基于mksaas-template（Next.js + PostgreSQL + Better Auth + Stripe + Resend）

## 设计要求
1. 按照EARS格式编写验收标准（使用WHEN/THE/SHALL/IF/WHILE等关键词）
2. 功能按P0/P1/P2优先级分类
   - P0：MVP必需功能
   - P1：增强功能
   - P2：优化功能
3. 分4个阶段实施
   - 阶段0：MVP验证（1-2周）
   - 阶段1：智能化增强（2-3周）
   - 阶段2：商业化闭环（2周）
   - 阶段3：规模化优化（持续）
4. 每个阶段包含：时间估算、功能清单、验收标准、技术栈
5. 覆盖7个设计层面：
   - 前端层（UI/UX、组件库）
   - 后端层（API、数据库）
   - 交互层（状态管理、API契约）
   - 中间件层（异步任务、缓存）
   - SEO/AIO层（搜索优化、AI索引）
   - 部署层（环境变量、CI/CD）
   - 安全层（认证授权、数据保护）

## 输出格式
请按照以下章节输出完整文档：

### 1. 项目概述
- 1.1 业务背景（问题陈述、解决方案）
- 1.2 目标用户（用户画像、使用场景）
- 1.3 核心价值主张（差异化优势）
- 1.4 成功指标（DAU、转化率、收入等）

### 2. 功能需求
- 2.1 核心功能列表（按P0/P1/P2分类）
- 2.2 每个功能包含：
  - 用户故事（作为[角色]，我希望[功能]，以便[价值]）
  - 验收标准（EARS格式，5-8条）
  - 技术实现（前端、后端、依赖、风险）
  - 验证方式（测试、监控）

### 3. 技术架构
- 3.1 技术选型（复用mksaas-template的能力 + 新增技术栈）
- 3.2 系统架构图（用Mermaid或文字描述）
- 3.3 数据模型设计（Drizzle Schema格式）
- 3.4 API设计（端点、Request/Response）

### 4. 非功能需求
- 4.1 性能要求（响应时间、并发数）
- 4.2 安全要求（认证、授权、加密）
- 4.3 可观测性（日志、监控、告警）
- 4.4 SEO/AIO优化（Meta标签、Sitemap）

### 5. 实施计划
- 5.1 阶段0：MVP（时间、目标、功能清单、验收标准、技术栈）
- 5.2 阶段1：增强（同上）
- 5.3 阶段2：商业化（同上）
- 5.4 阶段3：规模化（同上）

### 6. 风险管理
- 6.1 技术风险（风险、影响、概率、应对策略）
- 6.2 业务风险（同上）
- 6.3 应对策略总结

### 7. 成本预估
- 7.1 开发成本（人力、第三方服务）
- 7.2 运营成本（月度，列出所有服务费用）
- 7.3 ROI分析（收入预测、回本周期）

### 8. 附录
- 8.1 参考资料
- 8.2 竞品分析
- 8.3 技术调研

请开始设计，输出完整的需求文档。
