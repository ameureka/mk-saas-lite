本指南帮助你在2小时内完成一份高质量的需求文档。

| 3. 技术架构 | 30分钟 | 技术选型 + 数据模型 |

## 第1步：项目启动（5分钟）

### 目标
明确项目的基本信息和核心问题

### 操作步骤

1. **复制模板**
   - 打开 `templates/项目启动模板.md`
   - 复制整个模板

2. **填写基本信息**
   ```markdown
   项目名称：AI智能PPT模板推荐平台
   核心问题：用户需要花费大量时间浏览和筛选PPT模板
   目标用户：职场人士、教师、学生
   竞品参考：第一PPT、包图网
   技术基础：mksaas-template
   ```

3. **发送给AI**
   - 将填好的模板发送给AI（ChatGPT/Claude）
   - 等待AI生成初稿（约2分钟）

### 输出物
- 项目概述章节的初稿

---

## 第2步：功能列举（30分钟）

### 目标
列出所有功能并编写用户故事和验收标准

### 操作步骤

1. **列出核心功能**（5分钟）
   ```markdown
   核心功能：
   1. 智能语义搜索
   2. 模板详情展示
   3. 下载权限管理
   4. 用户积分系统
   5. 邮箱营销系统
   ```

2. **为每个功能编写用户故事**（10分钟）
   - 使用 `templates/功能需求模板.md`
   - 格式：作为[角色]，我希望[功能]，以便[价值]
   
   ```markdown
   功能1：智能语义搜索
   用户故事：作为用户，我希望通过自然语言描述找到合适的PPT模板，
   以便快速获取符合需求的资源
   ```

3. **编写EARS格式验收标准**（15分钟）
   - 每个功能5-8条验收标准
   - 使用WHEN/THE/SHALL/IF/WHILE关键词
   
   ```markdown
   验收标准：
   1. WHEN User输入自然语言查询，THE Platform SHALL生成embedding向量
   2. THE Platform SHALL在3秒内返回搜索结果
   3. IF 搜索无结果，THEN THE Platform SHALL显示推荐模板
   ```

### 快捷方法
- 让AI帮你生成验收标准
- 使用 `02-提示词模板集.md` 中的"功能需求细化提示词"

### 输出物
- 功能需求章节（包含用户故事和验收标准）

---

## 第3步：技术架构（30分钟）

### 目标
确定技术选型、设计数据模型和API

### 操作步骤

1. **技术选型对比**（10分钟）
   - 列出需要新增的技术
   - 对比3个候选方案
   - 说明选择理由
   
   ```markdown
   需要新增：向量数据库
   
   | 方案 | 优点 | 缺点 | 成本 | 推荐 |
   |------|------|------|------|------|
   | Pinecone | 免费层5M | 付费$70/月 | MVP:$0 | ✅ |
   | Qdrant | 开源 | 需运维 | 自建:$50 | 增长阶段 |
   | Milvus | 功能强 | 复杂 | 自建:$100 | 规模化 |
   
   选择：Pinecone（免费层足够MVP，无需运维）
   ```

2. **设计数据模型**（10分钟）
   - 用Drizzle Schema格式
   - 定义所有表结构
   - 说明表之间的关系
   
   ```typescript
   export const templates = pgTable('templates', {
     id: uuid('id').primaryKey().defaultRandom(),
     title: text('title').notNull(),
     embeddingId: text('embedding_id'),
     createdAt: timestamp('created_at').defaultNow(),
   });
   ```

3. **设计API**（10分钟）
   - 列出所有端点
   - 定义Request/Response格式
   
   ```typescript
   POST /api/search
   Request: { query: string, limit?: number }
   Response: { results: Template[], total: number }
   ```

### 快捷方法
- 使用 `templates/技术架构模板.md`
- 让AI生成数据模型和API设计

### 输出物
- 技术架构章节（选型、数据模型、API）

---

## 第4步：阶段规划（20分钟）

### 目标
将功能分配到4个阶段，每个阶段可独立交付

### 操作步骤

1. **功能分类**（5分钟）
   - P0：MVP必需功能
   - P1：增强功能
   - P2：优化功能
   
   ```markdown
   P0：搜索、详情、下载
   P1：积分、邮箱、广告
   P2：推荐、管理后台
   ```

2. **阶段分配**（10分钟）
   - 阶段0（MVP）：只做P0功能
   - 阶段1（增强）：P0优化 + 部分P1
   - 阶段2（商业化）：剩余P1
   - 阶段3（规模化）：P2 + 优化
   
   ```markdown
   阶段0：MVP（1-2周）
   - [ ] 基础搜索
   - [ ] 详情页
   - [ ] 简单下载
   验收：DAU>50
   
   阶段1：增强（2-3周）
   - [ ] AI语义搜索
   - [ ] 相似推荐
   验收：搜索准确率>80%
   ```

3. **时间估算**（5分钟）
   - 每个功能估算工作量
   - 每个阶段不超过3周
   - 预留20%缓冲时间

### 快捷方法
- 使用 `templates/阶段规划模板.md`
- 参考 `01-方法论.md` 中的阶段划分原则

### 输出物
- 实施计划章节（4个阶段的详细规划）

---

## 第5步：风险评估（15分钟）

### 目标
识别主要风险并制定应对策略

### 操作步骤

1. **识别风险**（5分钟）
   - 技术风险：技术选型、性能、第三方依赖
   - 业务风险：用户增长、变现、竞品
   - 团队风险：人员、技能、进度
   
   ```markdown
   技术风险：
   - 向量数据库成本超预算
   - LLM API限流
   
   业务风险：
   - 用户增长慢
   - 内容版权问题
   ```

2. **评估影响和概率**（5分钟）
   - 影响：高/中/低
   - 概率：高/中/低
   - 优先处理：高影响+高概率
   
   ```markdown
   | 风险 | 影响 | 概率 | 优先级 |
   |------|------|------|--------|
   | 成本超预算 | 高 | 中 | 高 |
   | 用户增长慢 | 高 | 高 | 高 |
   ```

3. **制定应对策略**（5分钟）
   - 预防措施：如何降低概率
   - 应急预案：如果发生了怎么办
   
   ```markdown
   风险：成本超预算
   应对：
   1. 初期使用免费层
   2. 超出后迁移到自建Milvus
   3. 设置预算告警
   ```

### 快捷方法
- 使用 `templates/风险评估模板.md`
- 参考 `04-常见陷阱.md` 中的常见风险

### 输出物
- 风险管理章节（风险清单 + 应对策略）

## 第6步：成本预估（15分钟）

### 目标
估算开发和运营成本，计算ROI

### 操作步骤

1. **开发成本**（5分钟）
   - 人力成本：人数 × 周数 × 周薪
   - 第三方服务：域名、测试费用
   
   ```markdown
   开发成本：
   - 人力：1人 × 4周 × $500 = $2000
   - 域名：$12/年
   - 总计：$2012
   ```

2. **运营成本**（5分钟）
   - 列出所有第三方服务
   - 区分免费层和付费层
   - 预测不同阶段的成本
   
   ```markdown
   | 服务 | MVP | 增长 | 规模化 |
   |------|-----|------|--------|
   | Vercel | $0 | $20 | $20 |
   | Pinecone | $0 | $70 | 自建$50 |
   | 总计 | $10 | $216 | $361 |
   ```

3. **ROI分析**（5分钟）
   - 预测收入
   - 计算利润
   - 估算回本周期
   
   ```markdown
   中性场景：
   - 月收入：$300
   - 月成本：$216
   - 月利润：$84
   - 回本周期：24个月
   ```

### 快捷方法
- 使用 `templates/成本预估模板.md`
- 给出3个场景（悲观/中性/乐观）

### 输出物
- 成本预估章节（成本表 + ROI分析）

---

## 第7步：质量检查（10分钟）

### 目标
确保文档质量，没有遗漏

### 操作步骤

1. **使用检查清单**（5分钟）
   - 打开 `03-检查清单.md`
   - 逐项检查
   - 标记✅或❌
   
   ```markdown
   完整性检查：
   ✅ 明确了目标用户
   ✅ 定义了成功指标
   ✅ 覆盖了7个设计层面
   ❌ 部分功能缺少验收标准 → 需要补充
   ```

2. **修复问题**（5分钟）
   - 优先修复高优先级问题
   - 补充缺失的内容
   - 优化不清晰的描述

3. **最终检查**
   - 文档结构完整
   - 格式统一
   - 无明显错误

### 输出物
- 高质量的完整需求文档

## 常见问题

### Q1：2小时真的够吗？

A：够的，前提是：
- 使用提示词模板让AI辅助生成
- 专注于核心内容，不纠结细节
- 第一次可能需要2.5小时，熟练后1.5小时

### Q2：如果时间不够怎么办？

A：可以分两轮：
- 第一轮（1小时）：完成1-4步，生成MVP版本
- 第二轮（1小时）：完成5-7步，补充风险和成本

### Q3：必须严格按照这个流程吗？

A：不必须，但建议：
- 新手严格按照流程，避免遗漏
- 熟练后可以调整顺序
- 但7个步骤都不能省略

### Q4：如何提高效率？

A：3个技巧：
1. 使用AI辅助生成初稿
2. 复用之前项目的模板
3. 建立团队的需求库
