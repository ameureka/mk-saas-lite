# 阿里云 vs 腾讯云向量数据库对比

| 项目 | 阿里云灵积向量数据库 | 腾讯云向量数据库（Vector DB） |
| --- | --- | --- |
| 部署形态 | Serverless/实例化服务，支持 VPC，原生 Milvus 接口 | 托管服务，兼容 Milvus/Qdrant API（部分地区可选），支持 VPC |
| 计费模式 | 按存储、计算、QPS 分级；提供按需与包年包月两种 | 按实例规格（内存/CPU/QPS）计费，可按小时或包年包月 |
| 免费/体验额度 | 提供体验实例（小规格，适合 POC） | 一般提供公测/体验额度（视活动而定） |
| API 兼容性 | 兼容 Milvus 2.x 协议；支持 REST/gRPC/SDK | 支持 Milvus 协议；部分场景可用自研 API，需确认版本 |
| 数据上限 | 官方描述支持 10 亿+ 向量；实例规格从 GB 到 TB | 针对百亿级场景设计，支持分片/高可用；具体规格按实例定价 |
| 生态集成 | 与阿里云 ACK/K8s、OSS、Log、监控深度集成 | 与腾讯云 SCF、TDSQL、COS、日志等联动方便 |
| 适合场景 | 国内合规要求高、已有阿里云资源、需要 Milvus 兼容 | 已在腾讯云部署、需要与腾讯CVM/COS等业务联动 |

**参考建议**
1. 若公司已有阿里云资源/VPC，或需要与阿里云 ACK/OSS 集成，优先考虑灵积向量库。
2. 若主要业务在腾讯云，或需要与 COS、TDSQL、StarRocks 等联动，可选择腾讯云向量数据库。
3. 两者目前均支持 Milvus 协议，可以在 SDK 层保持通用；建议在 `embedding_index` 表中记录 `provider`、`instance_id` 字段，便于未来迁移。
4. 对于预算敏感且没有强制云厂商约束，也可以在早期使用 Pinecone/Qdrant 免费层，后续再迁移到对应云厂商服务。

> 具体定价随地区和活动变化较大，建议在选型前通过所在云厂商控制台查看最新的规格/价格，并结合数据量（向量数 × 维度 × 4 字节）估算实际占用后再选实例。
